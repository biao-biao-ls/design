## 项目配置管理

### 配置文件位置

测试项目配置统一管理在：`types/wokwi-test-projects.ts`

### 配置结构

```typescript
export interface TestProject {
  id: string              // Wokwi 项目ID
  name: string            // 项目显示名称
  description: string     // 项目描述
  type: 'arduino' | 'esp32' | 'raspberry-pi-pico'  // 项目类型
  features: string[]      // 项目特性列表
  url?: string           // 项目URL（可选）
}
```

### 使用方式

```typescript
import { TEST_PROJECTS, getProjectForScenario } from '~/types/wokwi-test-projects'

// 获取基础测试项目ID
const basicProjectId = TEST_PROJECTS.BASIC_ARDUINO.id

// 根据测试场景获取项目ID
const projectId = getProjectForScenario('basic-embed')
```

## 项目创建指南

### 创建新测试项目

1. **访问 Wokwi 官网**: https://wokwi.com
2. **创建新项目**: 选择合适的模板（Arduino、ESP32等）
3. **配置电路**: 根据测试需求添加组件
4. **编写代码**: 实现测试所需的功能
5. **保存项目**: 获取项目ID
6. **更新配置**: 在 `types/wokwi-test-projects.ts` 中添加项目信息

### 项目要求

- **稳定性**: 项目应该能够稳定运行，不出现编译错误
- **功能性**: 项目应该包含足够的功能来验证集成特性
- **简洁性**: 项目应该尽可能简单，专注于特定的测试目标
- **文档化**: 项目应该有清晰的注释和说明

## 项目维护

### 定期检查

- **可访问性**: 确保所有项目ID仍然有效
- **功能性**: 验证项目功能是否正常
- **兼容性**: 检查项目与最新 Wokwi 版本的兼容性

### 更新流程

1. **识别需要更新的项目**
2. **在 Wokwi 中修改项目**
3. **测试修改后的功能**
4. **更新配置文件中的描述**
5. **更新相关文档**

### 备份策略

- **项目导出**: 定期导出项目文件作为备份
- **配置备份**: 保持配置文件的版本控制
- **文档同步**: 确保文档与实际项目保持同步

## 故障排除

### 常见问题

1. **项目加载失败**
   - 检查项目ID是否正确
   - 验证项目是否为公开状态
   - 确认网络连接正常

2. **功能异常**
   - 检查项目代码是否有编译错误
   - 验证电路连接是否正确
   - 确认组件配置是否合适

3. **性能问题**
   - 简化电路复杂度
   - 优化代码逻辑
   - 减少不必要的组件
