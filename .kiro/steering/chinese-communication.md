# 简体中文交流指导

## 语言要求

**重要：AI 助手必须全程使用简体中文与用户交流**

### 基本原则

1. **语言一致性**
   - 所有回复、解释、建议都必须使用简体中文
   - 技术术语可以保留英文，但需要提供中文解释
   - 代码注释应使用简体中文

2. **交流风格**
   - 使用专业但友好的语调
   - 保持简洁明了，避免冗长的表述
   - 使用适合中文表达习惯的句式结构

3. **技术文档**
   - 需求文档、设计文档、任务列表都使用简体中文
   - 变量名和函数名可以使用英文，但注释必须是中文
   - 错误信息和日志输出优先使用中文

### 术语对照

| 英文术语 | 中文术语 | 说明 |
|---------|---------|------|
| Requirements | 需求 | 功能需求文档 |
| Design | 设计 | 系统设计文档 |
| Tasks | 任务 | 实现任务列表 |
| Property-based testing | 基于属性的测试 | 测试方法 |
| User Story | 用户故事 | 需求描述方式 |
| Acceptance Criteria | 验收标准 | 需求验收条件 |

```typescript
  async validateCredentials(credentials: LoginCredentials): Promise<AuthResult> {
    // 实现认证逻辑
  }

```

### 文档结构

所有技术文档应遵循以下中文结构：

1. **需求文档**
   - 介绍
   - 术语表
   - 需求（包含用户故事和验收标准）

2. **设计文档**
   - 概述
   - 架构设计
   - 组件和接口
   - 数据模型
   - 正确性属性
   - 错误处理
   - 测试策略

3. **任务文档**
   - 概述
   - 任务列表（使用中文描述）

## 执行要求

- AI 助手在回复时必须检查语言使用是否符合此指导
- 如果用户使用英文提问，仍需用简体中文回复
- 保持技术准确性的同时确保中文表达的自然流畅
